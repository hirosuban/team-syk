# ARCHITECTURE ── ファイル構成

本リポジトリは、薄い技術書「そのIDは、人ではない」の原稿・検討ドキュメント群です。
執筆に直接関わるファイルの構成を示します（Re:VIEW/Grunt のビルドテンプレート類は割愛）。

## ルート

| ファイル | 役割 |
|---|---|
| `CLAUDE.md` | プロジェクト方針の正典。スコープ（2日で書く薄い本）・読者・「どんな本にするか」・執筆手法。目次の正典が `目次.md` であることを宣言する。 |
| `目次.md` | **確定した目次（唯一の正）**。全体構成を変えるときはこのファイルを更新する。序章＋第1〜4章、終章なし。 |
| `ARCHITECTURE.md` | このファイル。原稿・検討ドキュメントの構成図。 |

## docs/ ── 検討・執筆の作業ドキュメント

確定版ではなく、目次を固めるまでの検討過程・取材メモ・ドラフト。齟齬がある場合はルートの `目次.md` を優先する。

## draft/ ── 章ごとの本文下書き

[目次.md](目次.md) の各章に1対1で対応する、対話形式（聞き手＝下地／語り手＝山根）の本文下書き。書き方は `docs/chapter4_draft.md` を手本にしている。これを元に各自で執筆し、肉付けをしていく。

| ファイル | 対応する章 |
|---|---|
| `draft/序章.md` | 序章　我々は、受診勧奨という仕事をしている |
| `draft/第1章.md` | 第1章　全体の流れ ── 1通が届くまで |
| `draft/第2章.md` | 第2章　そのIDは、世帯であって、個人ではない（看板） |
| `draft/第3章.md` | 第3章　その番号は、左から何列目にいるか分からない |
| `draft/第4章.md` | 第4章　Excelか、CSVか |

## 正典の優先順位

1. `目次.md` ── 目次（何の章があるか）の唯一の正
2. `CLAUDE.md` ── プロジェクト方針の正
3. `docs/` 配下・`draft/` 配下 ── 検討過程・本文ドラフト（上2つと食い違ったら上を優先）
